# 종이 접기 3D 시뮬레이터 (paper-fold) — 설계 문서

날짜: 2026-07-21

## 목적

실물 종이(예: HAZE 커피 퍼즐 A4)는 한 번 접으면 자국이 남아 재사용이 어렵다.
앞·뒷면 이미지를 올린 가상의 종이를 3D 공간에서 자유롭게 접었다 폈다 하며
"접으면 앞뒤 그림이 겹쳐져 답이 드러나는" 류의 퍼즐을 무한 반복 실험할 수 있는 웹 도구.

## 확정된 요구사항

- 앞면/뒷면 이미지 업로드 (아무 이미지나 가능, 브라우저 안에서만 처리 — 서버 전송 없음)
- 종이 비율은 앞면 이미지 비율을 따름 (기본 A4)
- **자유 접기**: 아무 위치·각도로 선을 그어 접기 (미리 정의된 접선 없음)
- **평면 접기만** (180° 포개기). 종이비행기급 입체 접기는 범위 제외
- 슬라이더로 0~180° 접는 과정 미리보기, 180°에서 확정
- 앞으로 접기 / 뒤로 접기 (산접기/골접기) 모두 지원
- 여러 겹 접힌 상태에서 접으면 모든 층이 함께 접힘 (실물과 동일)
- 3D 회전/확대로 앞뒤 자유 관찰
- 되돌리기(한 단계), 완전히 펴기(리셋)
- 빛에 비추기 모드: 반투명 + 앞·뒷면 그림 곱셈 합성으로 겹침 확인

## 접근법 (선택: Three.js 자체 평면접기 엔진)

- 순수 HTML/CSS/JS + Three.js(CDN importmap), 빌드 도구 없음
- GitHub Pages 배포 예정 (다른 프로젝트들과 동일 패턴)
- 대안 검토: 2D 캔버스(3D 관찰 불가로 탈락), 기존 오리가미 시뮬레이터 개조(자유 접기·양면 텍스처 개조 비용 과다로 탈락)

## 기하 모델

- 종이 = 볼록 다각형 조각(facet)들의 목록. 초기엔 직사각형 1개
- facet = { pts: 종이좌표 다각형, T: 종이→테이블 2D 등거리변환(반사 포함), layer: 정수 }
- 접기 = 테이블 공간의 직선으로 모든 facet을 클리핑(Sutherland–Hodgman)
  → 접히는 쪽 조각은 T' = (직선 반사) ∘ T, 층 재배치
- 층 재배치: 위로 접기 = 접힌 더미가 남은 더미 위에 역순으로 쌓임
  (newLayer = maxStay + 1 + (maxFold − l)), 아래로 접기는 대칭 공식
- 직선 클리핑만 쓰므로 모든 facet은 항상 볼록 → 팬 삼각분할로 렌더링
- UV = 종이좌표 정규화 → 몇 번 접어도 이미지 유지. 뒷면 이미지는 로드 시 좌우 미러링
- det(T) 부호로 앞뒤 뒤집힘 자동 처리 (와인딩 반전 → FrontSide/BackSide 재질)
- 각도 미리보기: 접히는 조각들을 그룹으로 묶어 접선 축 회전(경첩 높이는 방향별 근사)

## UI

- 화면 전체 3D 뷰 + 상단 툴바: [앞면 이미지] [뒷면 이미지] [✂접기] [↩되돌리기] [⟲펴기] [💡비추기]
- 관찰 모드(기본): OrbitControls 회전/줌 (터치 지원)
- 접기 모드: 드래그로 선 긋기 → 접을 쪽 클릭 → 방향(앞/뒤) + 각도 슬라이더 → 180°에서 확정
- 상태별 안내 문구 표시

## 에러 처리

- 선이 종이를 지나지 않으면 안내 후 다시 긋기
- 미세 조각(면적 임계 이하)은 버림
- 이미지 로드 실패 시 기본 흰 종이 유지

## 테스트

- window.__fold 디버그 API(프로그램적 접기/되돌리기)로 브라우저 콘솔 검증
- 실물 HAZE 커피 종이 사진 2장으로 3번째 사진과 같은 접기를 재현해 확인
